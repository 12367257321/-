# 电网异常窃电智能识别系统 — 代码结构文档

## 项目概述

本项目是一个基于 **Flask + Oracle + Ollama AI** 的电网异常窃电智能识别系统。系统采用标准的三层架构，实现了用户用电数据管理、异常检测、窃电预警、AI 智能查询等核心功能。

## 核心文件详解

### 1. `app.py` — Flask 后端主程序（~1000行）

**作用**：系统后端核心，提供所有 HTTP API 服务和页面渲染。

**代码结构**：

| 模块 | 行号范围 | 功能 |
|---|---|---|
| **导入模块** | 1~45 | 导入 Flask、oracledb、ollama 等依赖库 |
| **NL2SQL 延迟加载** | 32~39 | 延迟加载 NL2SQL 引擎，避免启动时依赖缺失 |
| **数据库工具函数** | 52~180 | `get_conn()`、`query()`、`execute()`、`call_proc()` |
| **认证装饰器** | 183~227 | `@login_required`、`@admin_required` |
| **页面路由** | 234~299 | `/`、`/login`、`/user`、`/admin` 页面渲染 |
| **登录注册 API** | 300~399 | `/api/login`、`/api/register`、`/api/logout` |
| **概览 API** | 404~428 | `/api/overview` — 系统仪表盘统计 |
| **用户管理 API** | 437~510 | `/api/users` CRUD 操作 |
| **用电数据 API** | 517~629 | `/api/power` CRUD 操作 |
| **用电画像 API** | 638~700 | `/api/profile` — 画像分析 |
| **预警管理 API** | 705~810 | `/api/warnings` — 预警列表、处理、稽查 |
| **报表 API** | 815~860 | `/api/report` — 稽查报告生成与导出 |
| **规则配置 API** | 865~920 | `/api/rules` — 判定规则管理 |
| **社区动态 API** | 925~990 | `/api/posts` — 帖子、评论、点赞 |
| **VIP 管理 API** | 995~1050 | `/api/vip` — VIP 套餐管理 |
| **出远门管理 API** | 1055~1110 | `/api/absence` — 出远门登记 |
| **消息通知 API** | 1115~1160 | `/api/notifications` — 消息管理 |
| **AI 查询 API** | 1165~1200 | `/api/chat`、`/api/nlquery` — 自然语言查询 |
| **主函数** | 1205~1210 | `if __name__ == '__main__':` 启动服务 |

**核心模式**：每个 API 路由都遵循统一模式：

```python
@app.route("/api/xxx")
@login_required          # 登录验证
def api_xxx():
    try:
        data = query("SELECT ...")   # 查数据库
        return jsonify({"success": True, "data": data})
    except Exception as e:
        return jsonify({"success": False, "message": str(e)})
```

### 3. `nl2sql.py` — NL2SQL 引擎（RAG架构）

**作用**：将用户自然语言问题自动转换为 Oracle SQL 语句并执行查询。

**技术架构**：

```
用户中文问题 → 嵌入模型(bge-m3)向量化 → ChromaDB向量检索 → 拼接Prompt → 
→ Ollama大模型(qwen2.5:3b)生成SQL → SQL安全校验 → Oracle执行 → 返回结果 → AI解释
```

**核心类与方法**：

| 类/方法 | 功能 |
|---|---|
| `NL2SQLEngine` | NL2SQL 引擎主类 |
| `__init__()` | 初始化引擎，连接 ChromaDB |
| `query()` | 完整查询流程（同步模式） |
| `query_stream()` | 流式查询（SSE 模式） |
| `_search_schema()` | 向量检索相关 Schema 知识 |
| `_build_prompt()` | 构建 Prompt 模板 |
| `_validate_sql()` | SQL 安全校验（禁止 DDL/DML） |

**安全机制**：

- `FORBIDDEN_KW` 列表：禁止执行 `DROP`、`DELETE`、`INSERT`、`ALTER`、`TRUNCATE`、`CREATE` 等危险操作
- SQL 生成后自动检查，发现危险关键字直接拒绝执行

---

### 4. `rag_init.py` — RAG 知识库初始化（~250行）

**作用**：将数据库 Schema 知识向量化存入 ChromaDB，供 NL2SQL 引擎检索使用。

**知识库内容**：

| 文档类型 | 说明 |
|---|---|
| **表级描述** (`table_*`) | 每张表的字段、用途、关联关系 |
| **视图描述** (`view_*`) | 业务视图的定义和关键字段 |
| **示例 SQL** (`example_*`) | 常见查询的参考 SQL |
| **关键说明** (`key_note_*`) | 业务规则、判定逻辑等"软知识" |

**运行时机**：
- 首次部署系统时运行一次
- 数据库表结构或视图变更后重新运行

---

### 5. `realistic_data.py` — 仿真用电数据生成器（~200行）

**作用**：为演示账号生成高真实度的用电数据（25用户 × 90天 = 2250条记录）。

**真实度设计**：

| 设计要素 | 实现方式 |
|---|---|
| **工作日/周末差异** | 居民周末用电多，商业周末客流多 |
| **季节性差异** | 夏冬空调用电增加（季节系数 1.25~1.35） |
| **高斯噪声** | 正态分布随机扰动模拟真实波动 |
| **用户画像** | 三口之家、独居青年、购物中心、钢铁厂等不同用电模式 |

**窃电嫌疑注入**：

| 用户 | uid | 异常特征 |
|---|---|---|
| 钱七 | 5 | 后15天夜间骤降至正常的18% |
| 陈十三 | 6 | 后30天夜间骤降至15% |
| 杨二十 | 12 | 后20天夜间骤降至20% |
| 海底捞 | 19 | 后15天日间骤降至25%、夜间降至20% |
| 发电厂 | 25 | 后30天夜间骤降至12% |

---

### 6. `setup_oracle.py` — Oracle 一键初始化脚本（~550行）

**作用**：一键完成 Oracle 数据库的完整初始化。

**执行步骤**：

```
Step 1: 清理旧对象     → 删除表、序列、视图
Step 2: 建序列和表     → 4张核心表 + 4张辅助表 + 对应序列
Step 3: 插入规则和数据 → 3条判定规则 + 25个演示用户
Step 4: 生成用电数据   → 25用户 × 30天仿真数据
Step 5: 创建触发器     → 3个触发器（自动计算、实时预警）
Step 6: 创建视图和存储过程 → 3个业务视图 + 4个存储过程
收尾: 执行全量稽查     → 输出统计验证
```

## 数据库脚本详解

### 核心数据库对象

#### 表（11张）

| `risk_warn` | 预警记录 | `warn_id`、`user_id`、`warn_type`、`warn_level`、`is_handled` |
| `illegal_rule` | 判定规则 | `rule_id`、`night_ratio_threshold`、`fluctuation_threshold` |
| `community_post` | 社区帖子 | `post_id`、`user_id`、`content`、`create_time` |
| `post_comment` | 帖子评论 | `comment_id`、`post_id`、`user_id`、`content` |

#### 触发器（3个）

| 触发器 | 时机 | 功能 |
|---|---|---|
| `trg_power_bi` | BEFORE INSERT | 自动计算 `total_power` 和 `night_ratio` |
| `trg_power_ai` | AFTER INSERT | 实时检测夜间用电异常，生成预警 |
| `trg_power_bu` | BEFORE UPDATE | 更新时重新计算派生字段 |

#### 视图（3个）

| 视图 | 作用 |
|---|---|
| `v_user_power_profile` | 用户用电画像：统计日均用电、波动系数、夜间占比等 |
| `v_suspected_users` | 疑似窃电用户：双维度加权评分（满分60分） |
| `v_daily_trend` | 30天趋势：供前端折线图展示 |

#### 存储过程（6个）

| 存储过程 | 作用 |
|---|---|
| `sp_handle_warning` | 标记预警为已处理 |
| `sp_generate_report` | 生成疑似窃电稽查报告 |
| `sp_check_all_users` | 全量稽查：扫描所有用户生成预警 |
| `sp_system_overview` | 系统概览统计 |
| `fn_calc_risk_score` | 风险评分函数 |

## 数据流程图

### 实时预警流程

```
用户插入用电记录 → trg_power_bi(计算total_power和night_ratio) → 
→ user_power表存储 → trg_power_ai(检测异常) → 
→ night_ratio < 阈值? → 是 → 去重检查 → 插入risk_warn预警记录
```

### AI 查询流程

```
用户中文问题 → nl2sql.py → bge-m3向量化 → ChromaDB向量检索 → 
→ 拼接Prompt → qwen2.5:3b生成SQL → SQL安全校验 → Oracle执行 → 
→ 返回结果 + AI自然语言解释 → 前端展示
```

### 风险评分流程

```
user_power表 → v_user_power_profile(计算统计指标) → 
→ v_suspected_users(双维度评分) → 
→ 维度一(夜间占比40分) + 维度二(波动系数20分) = 总分0~60分 → 
→ 风险等级映射(一级/二级/三级)
```

## 运行命令

```powershell
# 1. 安装依赖
pip install -r requirements.txt

# 2. 初始化数据库
python setup_oracle.py

# 3. 下载 Ollama 模型
ollama pull bge-m3
ollama pull qwen2.5:3b

# 4. 初始化 RAG 知识库
python rag_init.py

# 5. 启动 Flask 服务
python app.py
```
