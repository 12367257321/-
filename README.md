# 电网异常窃电智能识别系统

基于 Oracle 21c + Python Flask 的电力反窃电稽查平台，采用 B/S 三层架构，通过多维度风险评分模型自动识别异常用电行为，并集成 NL2SQL 自然语言查询引擎，支持用中文直接查询数据库。

## 项目简介

本系统面向供电局稽查人员与用电用户，构建了从**数据存储 → 规则计算 → 异常预警 → 可视化展示 → 管理处理 → AI 查询**的完整闭环流程。系统当前承载 25 名注册用户、2250 条用电记录、992 条预警数据，在数据规模、对象类型完整性和功能覆盖面上均超出了课程设计的基本要求。

## 核心功能

### 管理员端（11个模块）

- **系统概览**：统计卡片 + 预警饼图，一键查看全局指标
- **用户管理**：用户档案增删改查
- **用电数据管理**：每日用电记录增删改查
- **用电画像分析**：11维画像指标 + 近30天趋势图
- **窃电预警管理**：一键全量稽查，标记处理预警
- **稽查报表**：疑似窃电用户排序 + Word 报表一键下载
- **AI 智能查询**：中文提问，自动生成 SQL 并流式返回结果
- **规则配置**：在线修改三类用户的判定阈值
- **出远门管理**：全局查看出远门记录
- **VIP 管理**：用户 VIP 状态与套餐管理
- **动态圈管理**：社区动态审核

### 用户端（7个模块）

- **我的用电**：个人用电画像 + 趋势图
- **用电趋势**：近30天日间/夜间用电对比
- **节电排名**：同类型用户排名榜单
- **省电动态圈**：发布心得、点赞、评论
- **出远门登记**：登记出行计划，避免误判
- **VIP 会员**：订阅会员套餐
- **AI 助手**：自然语言查询用电情况

## 核心算法

### 双维度加权风险评分

```
risk_score = 维度一评分 + 维度二评分（满分 60 分）

【维度一：夜间用电占比异常评分】（满分 40 分）
  avg_night_ratio < 0.10  → 40 分（严重异常）
  avg_night_ratio < 0.20  → 25 分（中等异常）
  avg_night_ratio >= 0.20 →  0 分（正常）

【维度二：用电波动性异常评分】（满分 20 分）
  fluct_coef > 0.60  → 20 分（严重波动）
  fluct_coef > 0.40  → 10 分（偏高波动）
  fluct_coef <= 0.40 →  0 分（正常）
```

### NL2SQL 自然语言查询引擎

基于 RAG（检索增强生成）架构，6 步流水线：

1. 用户中文问题 → BGE-M3 嵌入 → 1024维语义向量
2. ChromaDB 余弦相似度匹配 → top-5 相关 Schema 文档
3. 拼接 Prompt（检索文档 + 角色设定 + 字段速查表 + 硬约束 + 用户问题）
4. Ollama Qwen2.5:3B 生成 SQL → 正则提取纯 SQL
5. 安全检查（禁止 DROP/DELETE/INSERT 等）+ 自动修正列名错误
6. 执行 SQL → 取前20行 → LLM 自然语言解释结果

## 快速启动

### 1. 环境准备

- Python 3.9+
- Oracle 21c XE（已创建用户 `c##dbdesign` / `dbdesign123`）
- Ollama（已安装 `qwen2.5:3b` 和 `bge-m3` 模型）

### 2. 安装依赖

```bash
pip install flask oracledb python-docx chromadb ollama
```

### 3. 初始化数据库

```sql
-- 在 Oracle 中执行
@database/02_tables.sql
@database/03_triggers_views.sql
@database/04_procedures.sql
@database/05_run.sql
```

### 4. 启动服务

```bash
python app.py
```

服务启动后访问 **http://localhost:5000**

### 5. 公网访问（可选）

```bash
# 新开一个终端
& 'D:\数据库课设\ngrok.exe' http 5000
```

ngrok 会生成一个公网 URL，首次访问需点击 **Visit Site** 跳过警告页。

## 创新点

1. **NL2SQL 自然语言查询**：供电局工作人员无需掌握 SQL，用中文即可查询数据库
2. **双维度加权风险评分**：夜间占比 + 波动系数交叉验证，避免单一指标片面性
3. **触发器驱动实时预警**：录入即检测，无需等待定时批处理
4. **出远门误判规避**：登记出行后排除该期间数据，避免正常行为被误判
5. **Word 报表自动生成**：稽查结果一键导出标准化文档
6. **社区互动与 VIP 体系**：堵疏结合，调动用户节能积极性
